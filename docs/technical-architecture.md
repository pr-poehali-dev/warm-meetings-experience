# Описание технической архитектуры программного обеспечения

## Общие сведения

Платформа представляет собой веб-приложение типа SPA (Single Page Application) с серверной бесконтейнерной (serverless) архитектурой. Система предназначена для организации и проведения мероприятий, управления банными комплексами, мастерами, записями и пользователями.

---

## Инфраструктурные программные средства

| Категория | Решение |
|-----------|---------|
| Базовая операционная система | Linux (Ubuntu Server LTS) |
| Система виртуализации | Не применяется (бессерверная архитектура) |
| Балансировщик нагрузки | Встроенный балансировщик платформы Poehali.dev (FaaS) |
| Контейнеризация | Не применяется (FaaS — Function as a Service) |
| Средства хранения данных | PostgreSQL 15 |
| Облачное хранилище | S3-совместимое хранилище (bucket.poehali.dev, CDN: cdn.poehali.dev) |
| Веб-сервер / прокси-сервер | Nginx (встроен в платформу) |
| Программная платформа | Node.js 20 LTS (фронтенд), Python 3.11 (бэкенд) |
| Сервер приложений | Vite Dev Server (разработка), статическая раздача (продакшн) |
| СУБД | PostgreSQL 15 (psycopg2, Simple Query Protocol) |

### Фронтенд (SPA)

- Тип: Single Page Application, полностью клиентская маршрутизация
- Сборка: Vite (production build — статические файлы HTML/JS/CSS)
- Взаимодействие с бэкендом: HTTP-запросы к FaaS-функциям через REST API
- Аутентификация: токен в `localStorage`, передаётся через заголовок `Authorization`
- UI-компоненты: shadcn/ui поверх Radix UI (доступность WCAG AA)
- Темизация: CSS-переменные + Tailwind CSS, поддержка светлой/тёмной темы

**Основные разделы приложения:**
- Публичная часть: мероприятия, бани, мастера, блог, страница организатора
- Личный кабинет пользователя: профиль, записи, настройки
- Кабинет организатора: управление мероприятиями, мастерами, расписанием
- Административная панель: управление пользователями, контентом, ролями

### Бэкенд (Serverless Functions)

- Платформа: FaaS (Function as a Service) — poehali.dev
- Язык: Python 3.11
- Точка входа каждой функции: `def handler(event: dict, context) -> dict`
- Протокол: HTTP/HTTPS, формат обмена — JSON
- Авторизация: сессионный токен или JWT, передаётся через заголовок `X-Authorization` (прокси преобразует из `Authorization`)
- CORS: настроен на всех функциях (`Access-Control-Allow-Origin: *`)

### База данных

- СУБД: PostgreSQL 15
- Драйвер: psycopg2 (Simple Query Protocol)
- Управление схемой: Flyway-совместимые миграции (V####__ формат, 47 миграций)
- Схема: `public` (настраивается через переменную окружения `MAIN_DB_SCHEMA`)

**Основные таблицы:**
- `users`, `user_sessions`, `user_roles`, `login_devices`
- `events`, `event_signups`, `event_co_organizers`, `event_pricing_lines`
- `masters`, `specializations`, `master_calendar`, `master_baths`
- `baths` (банные комплексы)
- `bookings`, `booking_time_blocks`
- `roles`, `role_applications`
- `organizer_requests`
- `blog_articles`
- `refresh_tokens`, `reset_tokens`, `email_verify_tokens`
- `audit_logs`

### Хранилище файлов (S3)

- Endpoint: `https://bucket.poehali.dev`
- Бакет: `files`
- CDN: `https://cdn.poehali.dev/projects/{PROJECT_ID}/bucket/...`
- Клиент: boto3 (AWS S3-совместимый API)
- Используется для: фотографий мастеров, медиа бань, загружаемых пользователями изображений

### Аутентификация и безопасность

- Сессионная аутентификация: токен в `user_sessions` (30 дней)
- JWT: access token (15 минут) + refresh token (30 дней) — для Яндекс/VK OAuth
- Пароли: bcrypt (cost factor >= 12)
- 2FA: TOTP (RFC 6238) через pyotp, совместим с Google Authenticator
- OAuth 2.0: Яндекс ID, VK Connect
- Аудит: таблица `audit_logs` фиксирует все критичные действия
- Защита от дубликатов: нормализация email (lower case) перед поиском/созданием

### Внешние интеграции

| Сервис | Протокол | Назначение |
|--------|---------|-----------|
| Яндекс OAuth | OAuth 2.0 | Авторизация пользователей |
| VK Connect | OAuth 2.0 | Авторизация пользователей |
| Telegram Bot API | REST/HTTPS | Уведомления и управление через бота |
| UniSender Go | REST/HTTPS | Транзакционная email-рассылка |

## Масштабирование и отказоустойчивость

- Фронтенд: статическая раздача через CDN, автоматическое масштабирование
- Бэкенд: FaaS (serverless) — автоматическое горизонтальное масштабирование по нагрузке, изоляция функций
- БД: управляемый PostgreSQL с резервным копированием на уровне платформы
- Файловое хранилище: S3-совместимое с CDN-репликацией
